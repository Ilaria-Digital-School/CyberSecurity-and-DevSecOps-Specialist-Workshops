# Identify errors in a frame

THEORETICAL RECAP An Ethernet frame is a structured unit of data that operates at Layer 2 (Data Link) of the OSI model. It carries a payload from one MAC address to another within the same network segment. A standard Ethernet frame contains the following fields in order: Preamble (7 bytes) + Start Frame Delimiter (1 byte), Destination MAC address (6 bytes), Source MAC address (6 bytes), EtherType or Length field (2 bytes), Payload/Data (46 to 1500 bytes), and FCS — Frame Check Sequence (4 bytes, a CRC checksum). Each field has a precise role. The FCS is calculated by the sender and recalculated by the receiver; any mismatch signals a corrupted frame. Common mistakes beginners make include confusing MAC and IP addresses, misreading hexadecimal values, overlooking byte order (big-endian), and ignoring minimum payload size constraints (padding is added if the payload is smaller than 46 bytes). A useful mental model: think of a frame as a physical envelope — it has a sender address, a recipient address, contents, and a tamper seal (FCS). If the seal does not match the contents, the envelope has been altered in transit.
