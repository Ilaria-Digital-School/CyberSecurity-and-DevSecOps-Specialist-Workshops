# Full traffic — from HTTP request to response (annotated diagram)

## Project Overview This mini-project is the capstone activity for the 'Fundamentals of OSI Networks, Protocols and Models' chapter. You will produce a fully annotated diagram tracing the complete lifecycle of an HTTP request — from the moment a user types a URL in a browser to the moment the HTTP response is rendered — covering every protocol layer and network mechanism involved. Building on your work with Wireshark, DNS resolution, NAT, iptables, subnetting, and OSI layers, you will now synthesize all of these concepts into a single, coherent, documented artifact. This is not a theoretical exercise: you will capture real traffic, analyze it, and use that capture as the factual basis for your diagram. ## Context As a future CyberSecurity & DevSecOps Specialist, understanding the exact sequence of events in an HTTP transaction is foundational. Every attack surface — DNS poisoning, MITM, session hijacking, injection — lives somewhere along this flow. Before you can pentest, correct, or secure an application, you must be able to read and reconstruct network traffic with precision. ## Scope of the Diagram The annotated diagram must cover, in order: 1. **DNS resolution** — recursive query chain, UDP/TCP on port 53, response with resolved IP 2. **TCP three-way handshake** — SYN / SYN-ACK / ACK between client and server 3. **NAT translation** — how the client's private IP/port is rewritten at the gateway, iptables MASQUERADE or DNAT rules 4. **HTTP request** — method, headers, host, path, OSI layer mapping (L7 application, L4 TCP, L3 IP,...
