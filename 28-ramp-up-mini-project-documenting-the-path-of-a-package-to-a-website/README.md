# Ramp-up mini-project: documenting the path of a package to a website

## Project Overview In this mini-project, you will consolidate everything you have learned about OSI layers 1 through 3 by tracing, capturing, and documenting the complete journey of a network packet from your machine to a public website. This is not a theoretical exercise — you will use real tools, capture real traffic, and produce a professional technical document that mirrors what a cybersecurity analyst or DevSecOps specialist would deliver. This project sits at the intersection of network fundamentals and security analysis: understanding how packets travel across a network is a prerequisite for recognizing anomalies, performing network reconnaissance (e.g., with Nmap), and interpreting raw traffic in tools like Wireshark. --- ## Skills Validated - Identifying and describing the role of OSI layers 1, 2, and 3 in a real-world data transmission scenario - Reading and interpreting Ethernet frames (Layer 2: MAC addresses, frame structure) - Analyzing IP packets (Layer 3: source/destination IP, TTL, routing hops) - Using network diagnostic tools: `ping`, `traceroute`/`tracert`, Wireshark - Detecting and explaining potential anomalies or security-relevant observations in captured traffic - Producing structured technical documentation --- ## Context You are a junior cybersecurity analyst onboarding at a DevSecOps team. Your team lead asks you to produce a network path report for a target website of your choice (e.g., `https://www.wikipedia.org`) as a baseline documentation task before any penetration testing engagement. This report will serve as a reference for understanding the normal traffic behavior of an asset.
