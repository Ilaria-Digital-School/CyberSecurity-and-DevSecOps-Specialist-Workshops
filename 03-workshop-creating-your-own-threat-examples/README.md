# Workshop: Creating your own threat examples

THEORETICAL RECAP Before diving into the workshop, let's consolidate what you have covered so far. Four core concepts form the backbone of cybersecurity thinking: security (the overall state of protection of systems and data), threat (any potential event or actor that could cause harm), vulnerability (a weakness in a system that can be exploited), and risk (the combination of likelihood and impact when a threat meets a vulnerability). A simple mental model: a threat is the danger lurking outside, a vulnerability is the unlocked door, and risk is what happens when the two meet — the actual exposure your organisation faces. The CIA triad adds a second layer: Confidentiality ensures only authorised parties access data; Integrity ensures data is not tampered with; Availability ensures systems are accessible when needed. Every real-world security incident can be mapped to at least one of these pillars. Common mistakes at this stage: confusing threat with vulnerability (a weak password is a vulnerability; a brute-force attack is the threat), or treating risk as binary (present/absent) rather than as a spectrum. Also, learners often forget that a single incident can simultaneously breach multiple CIA pillars — a ransomware attack destroys availability AND integrity. Mini example: A public API endpoint that returns full user records without authentication is a vulnerability (missing access control). A malicious actor scanning the internet for exposed APIs is a threat.
