# Mini-project: create a YAML mini-pipeline with commented security steps

## Project Overview This mini-project is the capstone activity of the INTRO chapter. Having covered Git security policies, pre-commit hooks, OWASP Top 10, CIA triad, Shift-Left/Shift-Right/Feedback Loop principles, and pipeline security mapping, you will now consolidate everything into a single, concrete deliverable: a functional YAML CI/CD mini-pipeline that embeds security steps at the right stages and is fully annotated to demonstrate your understanding of why each step exists. This is not a theoretical exercise. You will write real YAML (GitHub Actions, GitLab CI, or any equivalent runner of your choice), integrate at least three distinct security tool categories (SAST, SCA, secret scanning), and justify every decision in inline comments and a companion document. ## Why This Project Matters In a DevSecOps role, the ability to translate security concepts into pipeline configuration is a core daily skill. Employers and clients evaluate DevSecOps engineers not only on their knowledge of vulnerabilities but on their capacity to automate security gates that catch those vulnerabilities before code reaches production. This project is your first proof of that capacity. ## Skills Being Validated 1. Translate Shift-Left principles into concrete pipeline stage ordering. 2. Map OWASP Top 10 risks to appropriate automated checks within a pipeline. 3. Configure and comment security-oriented CI/CD steps (secret scanning, SAST, SCA). 4. Apply secure branch policies and pre-commit concepts at the pipeline level. 5. Produce clear technical documentation explaining security choices to both technical and non-technical stakeholders. ## Scope and Constraints - The pipeline must target a simple web application...
