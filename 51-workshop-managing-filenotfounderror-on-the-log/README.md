# Workshop: managing FileNotFoundError on the log

**Theoretical Recap** Exception handling in Python is a fundamental mechanism that allows your program to react gracefully when something unexpected happens at runtime, instead of crashing abruptly. The core syntax revolves around the try/except block: you place the risky code inside the try block, and you define what to do if a specific error occurs inside the except block. Python has a rich hierarchy of built-in exceptions; one of the most common when working with files is FileNotFoundError, which is raised when you attempt to open a file that does not exist at the specified path. A common mistake beginners make is catching all exceptions with a bare except clause (except:), which hides bugs and makes debugging very difficult. Always aim to catch the most specific exception relevant to your situation. Another frequent error is forgetting to close the file after opening it — using the with open(...) as f: pattern solves this automatically. Mental model: think of try/except as a safety net beneath a tightrope walker. The try block is the walk; the except block is the net that catches a specific type of fall. You can also add an else block (runs if no exception occurred) and a finally block (always runs, useful for cleanup). Mini example: ```python try: with open('access.log', 'r') as f: content = f.read() except FileNotFoundError: print('Error: the log file was not found. Please check the path.') ``` This pattern is exactly what you will apply and extend in this workshop. --- **Workshop Brief** In...
