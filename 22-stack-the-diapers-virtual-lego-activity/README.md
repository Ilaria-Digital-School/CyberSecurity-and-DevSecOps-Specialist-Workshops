# Stack the diapers (virtual Lego activity)

**Theoretical Recap** The OSI model (Open Systems Interconnection) is a conceptual framework that standardizes network communication into 7 distinct layers: Physical (1), Data Link (2), Network (3), Transport (4), Session (5), Presentation (6), and Application (7). Each layer has a specific role and communicates only with the layers directly above and below it. The critical concept to internalize here is encapsulation: when data travels down the stack from layer 7 to layer 1, each layer wraps the payload from the layer above with its own header (and sometimes a trailer), forming what is called a Protocol Data Unit (PDU). At layer 7, you have data. At layer 4, a segment (TCP) or datagram (UDP). At layer 3, a packet. At layer 2, a frame. At layer 1, bits. A common mistake is confusing the role of the Network layer (IP addressing, routing) with the Data Link layer (MAC addressing, local delivery). Another frequent error is forgetting that encapsulation is reversed on the receiving end — de-encapsulation strips headers layer by layer going upward. A mini example: when you send an HTTP request, the browser (L7) generates data, TCP (L4) adds a port header, IP (L3) adds source/destination IPs, Ethernet (L2) adds MAC addresses, and the physical layer (L1) converts everything to a signal.
