# Scan ports with Nmap (beginner level)

**Theoretical Recap** A port is a logical endpoint at the transport layer (Layer 4) that allows a host to run multiple network services simultaneously — this is the multiplexing concept you studied in the previous lesson. When you connect to a web server on port 80 or 443, or query DNS on port 53, your operating system is directing traffic to the right service using that port number. Nmap (Network Mapper) is an open-source reconnaissance tool that probes a target host or network to discover which ports are open, filtered, or closed, and which services and versions are running behind them. It operates primarily at Layers 3 and 4, sending crafted TCP/UDP packets and analyzing responses. A common beginner mistake is confusing the three port states: **open** (a service is actively listening), **closed** (the port is reachable but no service is listening), and **filtered** (a firewall or rule is blocking probe packets, so Nmap cannot determine the state). Another frequent error is running aggressive scans (-A or -T5) on targets without authorization — always work on explicitly authorized environments. A minimal mental model: Nmap sends a SYN packet to a port, receives a SYN-ACK if open (completing the TCP handshake knowledge you practiced with netcat), a RST if closed, or silence/ICMP unreachable if filtered.
