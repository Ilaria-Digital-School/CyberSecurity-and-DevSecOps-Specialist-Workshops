# Adjust the subnetwork for a need

**Theoretical Recap** A subnet (subnetwork) is a logical subdivision of an IP network. The core tool for defining subnets is CIDR (Classless Inter-Domain Routing) notation, written as an IP address followed by a prefix length (e.g., 192.168.1.0/24). The prefix length tells you how many bits are reserved for the network portion, leaving the rest for hosts. A /24 gives you 256 addresses (254 usable hosts), a /26 gives 64 addresses (62 usable hosts), a /28 gives 16 addresses (14 usable hosts), and so on. The fundamental formula is: usable hosts = 2^(32 - prefix) - 2 (subtracting network and broadcast addresses). A common mental model: think of the prefix as a zoom level on a map. The larger the prefix number, the more zoomed in you are — fewer hosts, smaller network. The most frequent mistakes beginners make are forgetting to subtract 2 from the host count, confusing the network address with the first usable host, and choosing a subnet that is either too large (wasting addresses) or too small (blocking future growth). Mini example: a team of 30 workstations needs its own subnet. A /26 (62 usable hosts) fits perfectly — a /27 (30 usable hosts) is technically exact but leaves zero room for growth, while a /24 wastes over 200 addresses.
