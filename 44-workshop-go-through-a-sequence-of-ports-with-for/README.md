# Workshop: go through a sequence of ports with for

**Theoretical Recap** A `for` loop in Python iterates over a sequence — a range of numbers, a list, or any iterable object — and executes a block of code for each element. The built-in `range()` function is essential here: `range(start, stop, step)` generates a sequence of integers from `start` (inclusive) to `stop` (exclusive). For example, `range(1, 1025)` generates integers from 1 to 1024, which maps perfectly to the concept of reserved (privileged) ports in networking. In cybersecurity, ports below 1024 are considered system ports and require elevated privileges to open — a key concept you encountered in the previous workshop on port alerts. A common mistake beginners make is confusing `range(1, 1024)` (which stops at 1023) with `range(1, 1025)` (which correctly includes 1024). Another frequent error is forgetting that the loop variable changes at each iteration and must be used inside the loop body, not outside it. Mental model: think of the `for` loop as a conveyor belt — each port number arrives one by one, and your code inspects it before moving to the next. Mini example: ```python for port in range(1, 6): print(f'Checking port {port}') ``` This prints ports 1 through 5.
