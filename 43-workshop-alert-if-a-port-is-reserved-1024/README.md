# Workshop: alert if a port is reserved (<1024)

**Theoretical Recap** In networking, ports are numerical identifiers ranging from 0 to 65535 that allow operating systems to route incoming and outgoing traffic to the correct process or service. Ports below 1024 are called 'well-known' or 'reserved' ports. They are assigned by IANA (Internet Assigned Numbers Authority) to standardized services: port 22 for SSH, port 80 for HTTP, port 443 for HTTPS, port 21 for FTP, and so on. On Unix/Linux systems, binding to these ports typically requires root or administrator privileges, which makes them a frequent target for attackers and a critical point of attention in security audits. In Python, working with port numbers is a classic use case for variables and conditional logic — the two core concepts you have just studied. A port number is stored as an integer (int). A conditional check using if/else allows the program to branch its behavior depending on whether that integer is below 1024. A common beginner mistake is to compare a string to an integer without converting it first — for example, user input via input() always returns a string, so int() conversion is mandatory before any numerical comparison. Another mistake is inverting the comparison operator: to detect reserved ports, you want port < 1024, not port > 1024. Mini example: port = int(input('Enter a port number: ')) if port < 1024: print('Warning: this is a reserved port.') else: print('This port is in the user range.')
