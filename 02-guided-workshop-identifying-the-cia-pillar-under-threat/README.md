# Guided workshop: identifying the CIA pillar under threat

**Theoretical Recap** Before diving into the workshop, let's consolidate the essential framework you need. The CIA Triad is the cornerstone model of information security, composed of three pillars: Confidentiality, Integrity, and Availability. Confidentiality ensures that data is accessible only to authorized parties — a breach occurs when sensitive information is exposed to unauthorized individuals (e.g., a leaked password database). Integrity ensures that data remains accurate and unaltered by unauthorized actors — a breach occurs when data is modified without authorization (e.g., a hacker changing a bank transaction amount). Availability ensures that systems and data are accessible when needed — a breach occurs when legitimate users are denied access (e.g., a DDoS attack taking down a website). A common mistake beginners make is conflating confidentiality with integrity: remember, confidentiality is about WHO can see the data, integrity is about WHETHER the data has been tampered with. Another frequent error is forgetting that a single incident can threaten multiple pillars simultaneously — always analyze each pillar independently before drawing conclusions. A quick mental model: ask yourself three questions for every incident — 'Was private data exposed?' (Confidentiality), 'Was data altered?' (Integrity), 'Was access disrupted?' (Availability).
