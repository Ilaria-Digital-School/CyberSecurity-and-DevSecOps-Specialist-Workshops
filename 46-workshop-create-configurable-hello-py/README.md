# Workshop: create configurable hello.py

**Theoretical Recap** Python scripts become truly useful when they can adapt their behavior based on external input rather than running the same fixed logic every time. Command-line arguments, accessed via the `sys.argv` list, allow a script to receive parameters directly from the terminal when it is launched. `sys.argv[0]` is always the script name itself; actual user-provided arguments start at index 1. A common mistake is forgetting to check how many arguments were passed before trying to access them — accessing `sys.argv[1]` when no argument was given will raise an `IndexError` and crash your script. Always validate the length of `sys.argv` first using a conditional check (`if len(sys.argv) > 1`). Another frequent error is assuming arguments are the correct type: everything received via `sys.argv` is a string, so any numeric conversion must be done explicitly. Finally, Python scripts intended to be run directly from the terminal should include the shebang line (`#!/usr/bin/env python3`) at the top and be made executable with `chmod +x`. A minimal example: a script that prints a greeting to whoever is named on the command line (`python3 hello.py Alice` → `Hello, Alice!`), but falls back to a default message if no name is given. This pattern — validate, adapt, execute — is the foundation of every configurable automation script you will write in security contexts. **Workshop Brief** In this workshop, you will design and implement a configurable Python script named `hello.py`.
