# Guided workshop: associate each term with a concrete case

**Theoretical Recap** Before diving into the workshop, let's consolidate the four core vocabulary terms you encountered in the previous lesson. A **threat** is any potential event or actor that could cause harm to a system — for example, a hacker attempting to brute-force a login page. A **vulnerability** is a weakness in a system that could be exploited — such as a login page with no rate-limiting. A **risk** is the combination of the likelihood that a threat will exploit a vulnerability and the resulting impact — in this case, the risk is high if the application stores sensitive user data. **Security**, finally, refers to the set of measures put in place to reduce risks to an acceptable level. A common mistake beginners make is confusing threat with vulnerability: a threat is the *who* or *what* could cause harm; a vulnerability is the *flaw* that makes harm possible. Think of it this way — an unlocked window (vulnerability) in a bad neighborhood (threat environment) creates a burglary risk. Security means locking the window. Keep this mental model in mind throughout the workshop: **Threat + Vulnerability = Risk → Security measures reduce Risk**. This simple equation will help you reason clearly about each scenario you encounter below. --- **Workshop Brief** This workshop puts into practice the essential vocabulary introduced in the previous lesson.
