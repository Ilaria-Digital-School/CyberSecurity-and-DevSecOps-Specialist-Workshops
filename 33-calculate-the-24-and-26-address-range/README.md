# Calculate the /24 and /26 address range

**Theoretical Recap** An IP address in IPv4 is a 32-bit number divided into two parts: the network portion and the host portion. CIDR notation (Classless Inter-Domain Routing) expresses this split using a suffix like /24 or /26, which indicates how many bits are reserved for the network. The remaining bits define the available host addresses. For a /24 network: 32 - 24 = 8 bits for hosts. That gives 2^8 = 256 addresses total, with 254 usable hosts (the first address is the network address, the last is the broadcast address). For a /26: 32 - 26 = 6 bits for hosts, so 2^6 = 64 addresses, with 62 usable hosts. The subnet mask for /24 is 255.255.255.0 and for /26 is 255.255.255.192. A common mistake is forgetting to subtract the network and broadcast addresses when counting usable hosts, or miscalculating the block size when there are multiple /26 subnets inside a /24. Mini example: Given 192.168.1.0/26, the range is 192.168.1.0 to 192.168.1.63. Network: 192.168.1.0, Broadcast: 192.168.1.63, usable: .1 to .62. The next /26 block starts at 192.168.1.64. Keep this model in mind: block size = 2^(32 - prefix).
