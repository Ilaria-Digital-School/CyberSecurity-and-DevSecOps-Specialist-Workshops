# Workshop: Simulating a simple scheduled task

**Theoretical Recap** A scheduled task is a program or script that runs automatically at defined intervals or after a set delay, without human intervention. In cybersecurity and DevSecOps, scheduled tasks are everywhere: log rotation, automated vulnerability scans, credential expiry checks, port monitoring daemons. Understanding how to simulate them in Python is a foundational skill. The core mental model is simple: a loop (for or while) controls repetition, a sleep pause controls timing, and conditional logic controls what happens at each cycle. Python's time module provides time.sleep(seconds), which suspends execution for a given duration. Combined with a while True loop and a counter or condition, you can simulate a task that runs every N seconds, stops after M iterations, or halts when a specific condition is met. Common mistakes to avoid: forgetting to import the time module, using an infinite loop without any exit condition (causing the script to run forever with no way to stop cleanly), confusing time.sleep(1) (1 second) with time.sleep(1000) (nearly 17 minutes), and not printing any feedback — making it impossible to verify the script is actually running. Mini example: a script that checks every 5 seconds whether a port number variable has reached 0, printing a status message each cycle, then exits when the condition is true.
