# Workshop: est_pair (port) function

**Theoretical Recap** A function in Python is a reusable, named block of code that performs a specific task. You define it once using the `def` keyword, and call it as many times as needed throughout your script. Functions can accept parameters (inputs) and return values (outputs) using the `return` statement. This makes your code modular, readable, and easier to maintain — essential qualities in a security automation context where scripts are often reused across different targets and scenarios. A common mental model: think of a function as a machine. You feed it raw material (arguments), it processes it, and it outputs a result. For example, `def double(n): return n * 2` takes a number and returns its double. The modulo operator `%` is key here. `n % 2` returns 0 if `n` is even, and 1 if it is odd. This is a fundamental pattern used constantly in scripting and automation. Common mistakes to avoid: forgetting the `return` statement (the function then returns `None` by default), confusing `print()` with `return` (printing displays a value but does not make it available for further use in the code), and not testing edge cases such as 0 or negative numbers.
