# Observe an ICMP packet with ping

**Theoretical Recap** ICMP (Internet Control Message Protocol) is a network-layer protocol (OSI Layer 3) used primarily for diagnostic and error-reporting purposes. Unlike TCP or UDP, ICMP does not carry application data — it carries control messages. The most familiar use of ICMP is the `ping` command, which sends an ICMP Echo Request to a target host and waits for an ICMP Echo Reply. This round-trip exchange lets you verify reachability and measure latency. An ICMP packet is encapsulated inside an IP packet. The IP header contains source and destination addresses, TTL (Time To Live), and the protocol number 1 (reserved for ICMP). The ICMP header itself contains a Type field (e.g., Type 8 = Echo Request, Type 0 = Echo Reply), a Code field (usually 0 for ping), a Checksum, an Identifier, and a Sequence Number. The payload is typically a small block of padding bytes. A common beginner mistake is confusing ICMP with a transport protocol — ICMP has no port numbers and does not establish connections. Another frequent error is ignoring the TTL field, which is crucial in network security and traceroute logic. Keep in mind that firewalls often block ICMP, so a lack of reply does not always mean the host is down. Mini example: running `ping 8.8.8.8` sends ICMP Type 8 packets to Google's DNS server.
