# Workshop: Type conversions and simple operations

**Theoretical Recap** In Python, every value has a type: int (integer), float (decimal number), str (string of characters), bool (True/False). In real-world security scripting, data rarely arrives in the format you need. A port number read from a file comes as a string; a calculation result must sometimes be cast to an integer before being compared. This is where type conversion — also called casting — becomes essential. Python provides built-in functions to convert between types: int(), float(), str(), bool(). For example, int('443') produces the integer 443, and str(8080) produces the string '8080'. There are two kinds of conversion: implicit (Python does it automatically, e.g., adding an int and a float produces a float) and explicit (you do it deliberately with casting functions). Common mistakes to avoid: trying to convert a non-numeric string like int('abc') raises a ValueError — always validate input before converting. Also, bool(0) is False and bool('') is False, but bool('0') is True — a subtlety that trips up beginners. Arithmetic operators (+, -, *, /, //, %, **) behave differently depending on the type: '+' concatenates strings but adds numbers. Keep your mental model simple: types define what operations are possible; conversions let you move between types safely and deliberately. --- **Workshop Brief** This workshop builds directly on what you practiced in the previous session on declaring variables and displaying their types.
