# Workshop: reading a log and counting ERROR lines

**Theoretical Recap** In Python, reading a text file relies on the built-in `open()` function, which returns a file object. The standard pattern is to use `open()` in combination with a `with` block, which ensures the file is properly closed after reading, even if an error occurs. Once the file is open, you can iterate over its lines using a `for` loop, or load all content at once with `.read()` or `.readlines()`. Each line is a plain string, so you can apply string methods such as `.strip()` to remove trailing newline characters, `.startswith()` to check the beginning of a line, or `in` to check if a substring is present. In a security context, log files are a primary source of information. Being able to parse them programmatically is a foundational skill: you will regularly need to count occurrences of specific keywords (ERROR, WARNING, CRITICAL), extract timestamps, or isolate suspicious patterns. A common beginner mistake is forgetting to strip whitespace before comparing strings, leading to missed matches. Another pitfall is opening a file without a `with` block, risking resource leaks. A minimal example: ```python with open('app.log', 'r') as f: for line in f: if 'ERROR' in line: print(line.strip()) ``` This pattern — open, iterate, filter — is the mental model you should internalize before starting this workshop. --- **Workshop Brief** This workshop puts into practice the file-reading concepts introduced in the previous lesson "Reading text files: open, read", while building on your existing knowledge of functions, scripts, and command-line arguments.
