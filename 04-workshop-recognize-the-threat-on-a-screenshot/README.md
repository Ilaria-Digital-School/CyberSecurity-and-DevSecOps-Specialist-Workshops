# Workshop: recognize the threat on a screenshot

**Theoretical Recap** Before diving into the workshop, let's consolidate what you have covered so far. In cybersecurity, a threat is any potential event or action that could exploit a vulnerability to cause harm to an information system. A vulnerability is a weakness in a system, application, or process that a threat actor could leverage. Risk is the combination of the likelihood of a threat occurring and the impact it would have. These three concepts form the foundation of any security analysis. The CIA triad — Confidentiality, Integrity, and Availability — is your primary analytical lens. Confidentiality means only authorized parties can access data. Integrity means data remains accurate and unaltered by unauthorized parties. Availability means systems and data are accessible when needed. Every threat you encounter in the real world attacks at least one of these three pillars. Common web threats you have studied include phishing pages, SQL injection attempts, Cross-Site Scripting (XSS), brute-force login attempts, broken access control indicators, and exposed sensitive data. A common mistake beginners make is confusing the symptom visible on a screenshot with the root cause threat. For example, an error message revealing a database table name is a symptom — the underlying threat is SQL injection exploiting improper input validation. Always reason from what you observe toward what attack vector is being used and which CIA pillar is under threat.
