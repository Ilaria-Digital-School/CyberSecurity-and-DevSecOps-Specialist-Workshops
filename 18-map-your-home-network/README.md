# Map your home network

**Theoretical Recap** Before diving into the workshop, let's consolidate the core concepts you've encountered so far. A network is made up of hosts (any device with an IP address: computer, smartphone, printer, router, smart TV), links (the physical or wireless connections between those devices), and packets (the units of data exchanged across those links). Your home network is a real, live example of these abstractions. At the center sits your router/gateway, which connects your local network (LAN) to the internet (WAN). Every device on your LAN receives a private IP address (typically in the 192.168.x.x or 10.x.x.x range), while the router holds a single public IP address visible from the internet. A common beginner mistake is conflating the router with the modem: the modem handles the ISP connection, the router distributes it locally. Another frequent confusion is between IP addresses and MAC addresses — IP addresses are logical and can change; MAC addresses are hardware-burned identifiers. Keep this distinction sharp. A mental model that helps: think of your home network as an office building. The router is the reception desk (the only point of contact with the outside world), each room is a host, internal corridors are links, and envelopes circulating between rooms are packets. This workshop will make these concepts tangible by having you actively discover and document every component of your own network. --- **Workshop Brief** This workshop challenges you to produce a complete, documented map of your home network.
