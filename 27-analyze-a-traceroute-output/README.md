# Analyze a traceroute output

THEORETICAL RECAP Traceroute is a network diagnostic tool that maps the path packets take from your machine to a destination host, revealing each intermediate router (hop) along the way. It works by sending packets with incrementally increasing TTL (Time To Live) values. When a router decrements the TTL to zero, it sends back an ICMP 'Time Exceeded' message, exposing its IP address and round-trip time (RTT). On Linux/macOS the command is 'traceroute', on Windows it is 'tracert'. Each line in the output represents one hop: hop number, router IP (or hostname), and three RTT measurements in milliseconds. Mental model: think of traceroute as dropping breadcrumbs across a highway. Each tollbooth (router) reports back its identity and how long the detour took. Asterisks (*) mean the router did not respond — either it filters ICMP, or the packet was dropped. This does NOT necessarily mean the path is broken; traffic may still flow through silently. Common mistakes: confusing high RTT with packet loss; assuming * means a dead hop; ignoring asymmetric routing (return path may differ); forgetting that geolocation of IPs is approximate. A sudden RTT spike at one hop that then stabilizes at the next indicates that specific router deprioritizes ICMP responses, not a real bottleneck.
