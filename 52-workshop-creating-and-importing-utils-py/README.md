# Workshop: creating and importing utils.py

**Theoretical Recap** A module in Python is simply a .py file containing functions, variables, or classes that you can reuse across multiple scripts by importing it. When you write import utils or from utils import my_function, Python looks for a file named utils.py in the current directory (or in the directories listed in sys.path). This mechanism is fundamental to keeping code clean, readable, and maintainable — a principle known as DRY (Don't Repeat Yourself). A common mistake among beginners is placing utils.py in a different directory than the script that imports it, then being confused by a ModuleNotFoundError. Another frequent error is naming your file the same as a standard library module (e.g., logging.py), which shadows the built-in and causes unexpected import failures. Always name your utility files with specific, unambiguous names. Mental model: think of utils.py as your personal toolbox. Instead of rewriting the same log-parsing function in every script, you write it once in utils.py and import it wherever needed. For example: # utils.py def count_errors(lines): return sum(1 for line in lines if 'ERROR' in line) # main.py from utils import count_errors with open('app.log') as f: print(count_errors(f.readlines())) This is exactly the kind of refactoring you will practice in this workshop. --- **Workshop Brief** You have been working throughout this chapter with a log analysis script. So far, all your logic — opening files, counting ERROR lines, handling exceptions — has lived in a single script.
