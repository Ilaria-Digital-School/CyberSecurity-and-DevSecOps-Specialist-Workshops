# Use traceroute to see jumps

THEORETICAL RECAP Traceroute (or tracert on Windows) is a network diagnostic tool that maps the path packets take from your machine to a destination host, revealing each intermediate router — called a 'hop' — along the way. It operates at Layer 3 (Network) of the OSI model, leveraging the IP protocol's TTL (Time To Live) field. Each router that forwards a packet decrements the TTL by 1; when TTL reaches 0, the router discards the packet and sends back an ICMP 'Time Exceeded' message to the sender. Traceroute exploits this mechanism by sending packets with incrementally increasing TTL values (starting at 1), forcing each successive router to reveal itself. A common misconception is that traceroute shows a fixed, permanent path. In reality, packets can take different routes on successive runs due to load balancing, routing table changes, or ECMP (Equal-Cost Multi-Path) routing. Another frequent mistake is confusing high latency at a hop with a problem at that specific router — many routers deprioritize ICMP responses, making a hop appear slow even when the end-to-end path is healthy. Mini example: running 'traceroute 8.8.8.8' on Linux/macOS might show 10–15 hops, each with its IP address and round-trip time (RTT) in milliseconds. Asterisks (*) mean the router did not respond — not necessarily a failure, just a filtered ICMP response.
