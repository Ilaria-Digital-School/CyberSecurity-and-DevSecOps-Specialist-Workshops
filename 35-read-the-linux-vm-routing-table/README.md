# Read the Linux VM routing table

**Theoretical Recap** A routing table is a data structure stored in the kernel of an operating system that determines how network packets are forwarded from one network interface to another. Every Linux machine — whether a workstation, server, or virtual machine — maintains its own routing table, which the kernel consults for every outgoing packet. Each entry in the table typically contains: a destination network (expressed in CIDR notation, e.g., 192.168.1.0/24), a gateway (the next-hop IP address), a network interface (e.g., eth0, lo), and a metric (a cost value used to prefer one route over another when multiple routes exist). A common mental model: think of the routing table as a postal sorting guide. When a packet arrives at the kernel, it looks up the most specific matching destination (longest prefix match). If no specific route matches, it falls back to the default route (0.0.0.0/0), which points to the default gateway — typically your router. Common mistakes beginners make: confusing the gateway address with the destination network, misreading the CIDR prefix length (especially /24 vs /26, which you practiced in the subnetting module), and forgetting that the loopback interface (127.0.0.1/8) is always present and routes local traffic internally without leaving the machine. Mini example: a route entry like `192.168.10.0/24 via 0.0.0.0 dev eth0` means packets destined for any address in the 192.168.10.0–192.168.10.255 range are sent directly out of interface eth0 without going through a gateway (direct/local route).
